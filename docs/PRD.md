# 제품 기획서 (PRD)
**HR English Coach** — HR 담당자를 위한 비즈니스 영어 학습 웹앱

> 최종 수정: 2026-05-17 · 작성: MVP 출시 v1.0

---

## 1. 배경 & 문제 정의

### 1.1 사용자 페인포인트
- 한국 HR 담당자(HRBP/CHRO)는 영어로 CEO/임원과 전략 대화를 해야 하지만, 일반 영어 학습 콘텐츠는 비즈니스 맥락이 부족함
- 토익/오픽 학습은 회의실에서 쓰는 표현·논리 구조와 거리가 큼
- 1:1 영어 코칭은 비용이 크고 (월 수십~수백만원) 시간대 맞추기 어려움
- 자유 학습으로는 "오늘 무엇을, 어떤 깊이로" 정해서 꾸준히 하기 어려움

### 1.2 기존 솔루션의 한계
| 옵션 | 한계 |
|------|------|
| 화상 영어 (Cambly 등) | 비용 큼, HR 도메인 강사 드뭄 |
| 비즈니스 영어 책 | 인풋만 가능, 아웃풋(말하기) 연습 부족 |
| ChatGPT 자유 대화 | 시나리오 설계·반복 학습 구조 부재, 매번 프롬프트 작성 부담 |
| 토익/오픽 학원 | HR 직무 맥락 부재 |

## 2. 비전 & 목표

### 2.1 비전 문장
*"매일 20분, HR 담당자가 자신감 있게 영어로 전략 대화를 이끌도록 한다."*

### 2.2 MVP 목표 (Phase 1, 출시 시점)
- 사용자가 **혼자서 무료로** 비즈니스 영어 케이스를 매일 연습 가능
- **음성 듣기/말하기**로 회의 시뮬레이션 경험 제공
- **모범 답변과 비교**해 자가 피드백 가능
- **단어/용어 즉시 검색**으로 학습 단절 방지

### 2.3 성공 지표 (MVP 단계)
| 지표 | 목표 |
|------|------|
| 매일 1회 이상 사용 | 첫 30일 동안 18일 이상 (출석률 60%) |
| 평균 세션 시간 | 10~20분 |
| 케이스당 음성/텍스트 답변 제출 | 평균 3건 이상 |
| 자가 평가 만족도 | 5점 만점에 4점 이상 |

## 3. 타겟 사용자

### 3.1 Primary Persona — "수진" (37세, HRBP)
- 한국계 글로벌 IT 기업 HR Business Partner
- 영어 회의 주 1~2회, CEO/임원 대상 발표 분기당 1회
- 영어 자체는 의사소통 가능하지만 "비즈니스 격조 있는 표현"이 부족하다고 느낌
- 매일 출근 전 20분, 점심 시간 20분 활용 가능
- 학습 도구에 큰 비용 쓰기 어려움 (개인 부담)

### 3.2 Secondary Persona — CHRO/HR Director 직급
- 이미 영어가 능숙하지만, M&A·구조조정·DEI 등 고도 주제에서 더 정교한 표현을 갈고 닦고 싶음
- 시간이 매우 짧음 → 짧고 밀도 높은 콘텐츠 필요
- 고급 난이도 케이스, 정교한 어휘 피드백 선호

### 3.3 비타겟
- 토익/오픽 시험 대비 학습자
- 일상 회화 학습자
- 영어 초보 (왕초보) 학습자 — Beginner 난이도가 있지만 어느 정도 기초 있는 사용자 가정

## 4. 핵심 가치 제안

1. **HR 도메인 특화** — 일반 영어 앱이 다루지 않는 RIF, 보상, DEI, 노무 이슈
2. **무료, 키 없이 즉시 사용** — 브라우저만 있으면 됨, 부담 없는 시작
3. **음성 + 텍스트 양방향** — 출퇴근길은 음성, 집중 시간은 텍스트
4. **난이도 조절** — 같은 케이스를 3단계로 반복 학습 가능
5. **점진적 업그레이드** — 처음엔 무료, 필요 시 API 키 추가해 AI 피드백 강화

## 5. 범위 (Scope)

### 5.1 MVP (Phase 1, v1.0 — 출시 완료)
- ✅ 케이스 스터디 1개 (구조조정)
- ✅ 3단계 난이도 (Beginner / Intermediate / Advanced)
- ✅ 음성 듣기 (TTS, 속도 0.5~1.1x 조절)
- ✅ 음성으로 답변 (STT, Chrome/Edge)
- ✅ 텍스트로 답변
- ✅ 단어 클릭 → 사전 + 케이스 용어집
- ✅ 모범 답변 비교 + 답변 팁
- ✅ 학습 기록 (localStorage)
- ✅ 영어/한글 토글 표시
- ✅ 매일 추천 케이스/질문 자동 선택

### 5.2 Phase 2 (다음 분기)
- 케이스 스터디 10개 이상 (M&A, DEI, 보상 협상, 성과 관리, 노조 대응 등)
- AI 실시간 피드백 (Anthropic Claude API 또는 OpenAI API)
  - 문법 교정
  - 표현 업그레이드 제안
  - 답변의 논리 구조 코칭
- 고품질 음성 (Anthropic/OpenAI/ElevenLabs TTS)
- 매주 새 케이스 자동 생성 (Claude API + GitHub Actions)
- 학습 진척률 대시보드

### 5.3 Phase 3 (이후)
- 클라우드 학습 기록 (다기기 동기화) — Supabase 무료 티어
- 발음 평가 (Speechace API 등)
- 다른 사용자와 답변 비교 (선택적 익명 공유)
- 모바일 PWA / 네이티브 앱
- 한국어 외 언어 (일본어, 중국어 HR 영어 학습자)

### 5.4 비범위 (Out of Scope)
- 실시간 1:1 강사 매칭
- 비디오 콘텐츠
- 게임화 (배지/포인트) — 처음엔 단순함 유지
- 결제 시스템 — 무료 우선, 유료화는 검증 후

## 6. 핵심 사용자 시나리오

### 시나리오 A — 아침 출근 전 20분
1. 사용자 "수진"이 8:00 AM 카페에서 노트북을 연다
2. 북마크한 앱에 접속
3. "📅 오늘의 학습"이 자동으로 케이스/질문을 선택해 보여줌
4. 🔊 케이스 듣기 → 모르는 단어 클릭해서 의미 확인
5. 🎤 음성으로 답변 녹음 (1~2분)
6. 모범 답변과 비교, 답변 팁 확인
7. 같은 질문에 텍스트로 다시 답변 (정제된 표현 사용)
8. 학습 기록에 저장됨

### 시나리오 B — 분기별 임원 발표 직전 집중 연습
1. CHRO 후보가 발표 전주 매일 Advanced 난이도로 같은 케이스 반복
2. 처음엔 모범 답변 보고 → 다음엔 안 보고 답변 → 마지막엔 음성으로 즉답
3. 학습 기록으로 자신의 답변 변화 추적

### 시나리오 C — 새 케이스 자동 수신
1. (Phase 2) 매주 월요일 오전, GitHub Actions가 자동으로 새 케이스를 생성해 PR 등록
2. 운영자(또는 사용자 본인)가 PR 리뷰 후 머지
3. 다음 접속 시 새 케이스가 사용자의 학습 큐에 추가됨

## 7. 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| **성능** | 첫 로드 < 2초 (모든 자원 < 200KB) |
| **호환성** | Chrome/Edge: 모든 기능 / Safari: TTS 가능, STT 제한 / Firefox: TTS만 |
| **접근성** | 키보드 탐색 가능, 색상 대비 WCAG AA |
| **반응형** | 모바일 (375px) ~ 데스크톱 (1440px+) 정상 동작 |
| **프라이버시** | 학습 기록은 사용자 브라우저에만 저장. 외부 전송 없음 (사전 lookup 제외) |
| **비용** | Phase 1: 무료 / Phase 2: 사용량 기반 (월 $1~3 수준) |

## 8. 리스크 & 가정

### 리스크
- **저작권** — 케이스가 HBR 등 실제 출판물을 너무 가깝게 따라가면 안 됨 → 가상 회사명 사용, 시나리오 원작 창작
- **AI 생성 케이스 품질** — Phase 2에서 자동 생성 시 일관성 저하 가능 → 운영자 리뷰 단계 필수 (PR 모델)
- **사용자 습관 형성** — 매일 들어와야 의미 있음 → 리마인더(GitHub Issues), 오늘의 케이스 자동 선택 등 트리거 강화

### 가정
- 사용자는 영어 기초가 어느 정도 있다 (TOEIC 700+)
- 사용자는 GitHub Pages 또는 비슷한 정적 호스팅으로 접근 가능한 환경
- Phase 2 AI 기능은 사용자가 API 키 발급에 큰 거부감 없음 (또는 운영자가 키 제공)
